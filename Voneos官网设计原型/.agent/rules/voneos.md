1.**代码冲突预防规则**: 每次修改项目代码前,必须:
   - 查看相关文件的当前实现,理解现有逻辑
   - 检查相关的配置文件(如tailwind.config.js, tsconfig.json等)
   - 确认修改不会与现有代码产生冲突
   - 对于样式修改,优先检查是否已有相关CSS类可以复用
2.**错误记录与学习规则**: 当修改失败时,必须:
   - 在下方"错误记录"区域详细记录失败原因
   - 记录失败的文件路径、修改内容和错误信息
   - 分析失败原因(如:Tailwind类未生效、CSS优先级问题、ref绑定错误等)
   - 记录成功的解决方案,防止重复发生相同问题
3.**CSS文档组织规则**: 
   - 每个页面应该有独立的CSS文件(如: `home.css`, `brand.css`, `science.css`)
   - CSS文件应该与对应的组件文件夹在同一目录下
   - 在CSS文件中使用注释分区,例如:
     ```css
     /* ========================================
        Component Name Styles
        ======================================== */
     ```
   - 保持CSS类命名的语义化和一致性
4.**错误追踪规则**: 每次遇到错误或修改失败时,在下方"错误记录"区域按以下格式记录:
   ```
   [日期] [文件路径]
   问题: [问题描述]
   原因: [根本原因分析]
   解决方案: [成功的修复方法]
   ```
5.**首页样式管理规则**: 首页(Home Page)所有组件的样式属性优先在 `src/components/home/home.css` 中定义。当需要为首页元素添加样式时:
   - 首先检查是否可以使用home.css中已有的类
   - 如果没有合适的类,在home.css中创建新的CSS类
   - 避免在组件中使用过多的Tailwind内联类或内联style
   - 保持样式的集中管理和可维护性


## 错误记录

### 2026-01-07 产品卡片滚动功能失效
**文件**: `src/components/home/ProductShowcase.tsx`
**问题**: 点击左右导航按钮时,产品卡片无法滚动
**原因**: Tailwind的`overflow-x-auto`类在CSS产物中不存在,导致容器的`overflow-x`属性为`visible`而不是`auto`,使得`scrollBy`方法无法生效
**解决方案**: 在滚动容器的`style`属性中显式添加`overflowX: 'auto'`,确保样式生效
